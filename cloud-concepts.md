## What is Microsoft Azure?

- Azure is a cloud computing platform that allows you to use services such as hosting VM's, using remote storage and hosting databases. 

## What is Cloud Computing? 

- Cloud computing delivers services over the internet that are commonly used in IT infrastructure, such as databases, virtual machines, storage, and networks. 
- Businesses are not limited by physical infrastructure when using cloud services, they only pay for what they use and don't need to physically host a data center. 

---
    - Cloud computing allows for easy temp expansion to fit business needs that can be scaled down when no longer needed, such as a retailer needing more server capacity for a summer launch that slows down in the fall. 
  
    - This approach aligns well with an agile methodology, allowing services to be deployed and tested faster and easier, easily adapting to change as needed.

## What is the Shared Responsibility Model? 
- Some responsibilities surrounding the maintenance of a datacenter is on the cloud provider, while some is on the consumer. 
- Physical upkeep such as cooling, physical security, power and connectivity is the cloud providers responsibility
    - The data, information being stored in the cloud, and access security is the responsibility of the consumer, not as much the upkeep of the service itself

1. Information as a service (IaaS) - Most of the responsibility is on the consumer 
2. Platform as a service (PaaS) - Equal responsibilities between cloud provider and consumer, with some shared
3. Software as a service (SaaS) - Most of the responsibilities fall on the cloud provider

## What are the different Cloud Models? 
### Private Cloud 
- Allows for some of the benefits of cloud computing in special scenarios where public clouds are not appropriate, sacrificing cost and scalability
  - A business may choose private cloud computing if they are bound by legal requirements such as healthcare and government organizations (strict regulations on data)
  - They have security concerns and deal with highly sensitive information, or have already invested in expensive datacenters and hardware but want cloud benefits

### Public Cloud 
- Instead of all resources being allocated to one organization, public clouds trade control for more features, are hosted by a third party provider and allocate resources to whoever wants to use them 
- You pay for only what you use, there is no hardware costs, infinite scalability, not limited by location (all of these are factors to consider with private clouds)

### Hybrid Cloud
- Uses both private and public cloud methodologies in a single environment
- The most popular choice in most organizations, allows you to put sensitive or regulated workloads on the private cloud, while deploying adaptable resources, test environments, new applications to the public cloud

### Multicloud
- You use multiple public cloud providers, maybe using different features from each one, manage resources and security from both
